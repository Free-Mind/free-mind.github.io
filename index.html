javscript中的变量对象

Posted on 2017-09-04 | In javascript

本文主要记录的是执行上下文中的变量对象。

一、什么是作用域？

作用域是指代码中所定义的变量的可见范围
作用域分为词法作用域（静态作用域）和动态作用域
静态作用域：
变量的可见范围根据定义时的位置确定；
动态作用域：
变量的可见范围在代码执行时才能确定；
javascript采用的是静态作用域（词法作用域）

二、 变量对象的组成

变量对象中存储的是javascript代码执行上下文相关的数据作用域，也就是说，在运行时，javascript会在VO中寻找变量。VO中通常包含以下三种变量：

  • 通过var 关键字声明的变量；
  • 函数的声明
  • 函数的形参

注意：有两类变量不会出现在VO中：
① 没有用var关键字声明的变量，即全局变量
② 函数的表达式（非函数的定义）
例如，如果上述代码多了以下两行，则Global Execution Context不会改变：

没有使用var关键字声明的变量，其实就是window对象的一个属性，并不会存在VO中。

三、细看Execution Context

javascript代码的执行有两个阶段：
①进入执行上下文（函数内部代码执行之前）
②执行阶段，开始解释执行内部代码
我们首先看第一个阶段做了什么：
①创建作用域链（scope chain）
②创建VO对象
③创建this
这里详细的解释一下第二步：创建VO对象

根据函数形参创建arguments object；

所有的函数名和函数引用存入VO中；

遍历所有的函数声明，将函数名和函数引用存入VO中；
如果有重复的函数名，则后者覆盖前者；

变量声明，所有的变量收集到VO中；

遍历所有的变量声明，将函数名存入VO中，对应的value是undefined；
如果收集的变量名跟形参或者方法名重合，则变量名不会对其产生影响

javscript执行上下文

Posted on 2017-09-04 | In javascript

本文主要介绍的是javascript执行时的上下文概念。我们在学习javascript的时候，经常性的会被作用域，作用域链以及this这些概念搞混淆。其实这几个概念都跟javascript代码执行时的上下文有关，想要透彻的理解作用域链跟this，就必须深入的学习一下上下文对象：Execution context。

一、关键结论：

  • 三种代码执行环境：

    ①Global code
②Function Code
③Eval Code

  • 每个Execution Context 对象有三个属性：
    ① 变量对象（Variable Object）
    ② 作用域链（scope chain）
    ③ this

  • 变量对象包含三部分信息
    ① 变量
    ② 函数声明
    ③ 函数形参
    注意：函数的表达式以及没有通过var 关键字声明的变量，是不会出现在VO中的

一、代码执行环境

在javascript中，一般有三种代码执行环境：

  • Global Code
    javascript代码执行的第一个环境就是全局环境。全局环境中，this指向全局对象，在浏览器中就是window对象；

  • Function Code
    代码进入了一个函数中执行，这时候会创建一个新的执行上下文，而之前的Execution context对象会被压入堆栈的地步；

  • Eval Code
    在eval函数中执行的代码，本篇暂时不做介绍

最后进入innerFunc方法，还是Function Code环境，这时候会创建一个新的上下文：innerFuncExecution Context。这里，代码在每次进入一个新的执行上下文之前，旧的（也就是当前上下文）都会被压入“执行上下文堆栈”中被保存起来，如下图所示：

第一种方式中，函数的声明和定义都“被提前”了。所以，函数可以正确的被执行
第二种方式中，把函数作为值赋值给一个变量，变量的声明被提前，但是变量的赋值操作却不能够被提前，

“函数声明”式定义方法（方法①）：函数的声明和定义均被提前到作用于顶部；
“变量赋值”式定义方法（方法②）：函数的声明被提前到作用域顶部，但是赋值被留在原地。

spring MVC中的拦截器可以用来实现权限验证，判断用户是否登录等功能，接下来，我将会实现一个简单的拦截器。

1. 创建一个实现HandlerInterceptor 接口的类…

拦截器共有三个方法：

preHandle

postHandle

afterCompletion

这三个方法的执行顺序是：

preHandle -> postHandle -> afterCompletion

preHandle: 在请求进入controller的具体方法之前进行调用。一般用于权限验证；返回值：true：可以进入具体方法；false：禁止进入controller中的方法

postHandle： 在控制返回ModelAndView之前进行调用，可以修改ModelView的值

afterCompletion： 在视图渲染完成以后调用
